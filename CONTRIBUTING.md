## Процесс работы с кодом

### Коммиты
- Используйте **осмысленные имена** для коммитов, которые отражают суть изменения.
- Соблюдайте формат коммитов:

Структура:
```
<тип>(<опциональная_область>): <краткое_описание>

[опциональное_тело_коммита]

[опциональный_нижний_колонтитул]
```

Тип:
- **feat** - Добавление новой функции или возможности
- **fix** - Исправление ошибки в коде
- **docs** - Изменения только в документации
- **style** - Изменения, не влияющие на смысл кода (форматирование, пробелы и т.д.)
- **refactor** - Изменение кода, которое не исправляет ошибку и не добавляет новую функцию
- **test** - Добавление или исправление тестов
- **chore** - Обновление зависимостей, настройка проекта, задачи по сборке
- **ci** - Изменения в конфигурации CI/CD
